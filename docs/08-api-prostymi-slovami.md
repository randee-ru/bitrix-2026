# API сайта — простыми словами (для ученика)

Этот текст — **вторая ступенька**: если [07-legacy-api.md](07-legacy-api.md) кажется слишком «техническим», начните отсюда. Потом снова откройте `07` — там точные имена файлов и примеры кода.

---

## 1. Зачем вообще это нужно?

Сайт на Битрикс — это в основном **страницы в браузере** (HTML). Но иногда нужно, чтобы **программа** (JavaScript на странице, мобильное приложение, другой сервер) **получила данные в удобном виде**, чаще всего **в JSON** — как список товаров, статус заказа, «ок» или «ошибка».

Для этого делают **API** — по сути **отдельные адреса на сайте**, которые **не открывают красивую страницу**, а **отдают короткий ответ в машинном формате**.

У нас такие адреса начинаются с **`/api/`**.

---

## 2. Что происходит, когда кто-то открывает `/api/news`?

Представь **почтовое отделение**:

1. Человек (или скрипт) заходит по адресу: **`https://сайт.ru/api/news`**.
2. **Битрикс** смотрит свои правила: «О, путь начинается с `/api/`» — и **передаёт работу** не обычной странице, а **нашему маленькому PHP-файлу** — это вход в API.
3. Тот файл **узнаёт**, что после `/api/` написано **`news`**.
4. Дальше он смотрит в **табличку маршрутов** (в коде это `ApiRoutes`): есть ли строка **`/api/news`**?
   - **Есть** — вызывается **нужный класс** (например, новости), тот **готовит массив данных**.
   - **Нет** — отвечает заглушка: «такого метода нет» (как «нет такого ящика»).

5. В конце ответ **превращается в JSON** и **отправляется в браузер** — без шаблона сайта, без шапки и подвала.

То есть **один раз настроили таблицу «адрес → класс»**, и для каждого нового типа данных добавляете **новую строку** и **новый маленький класс**.

---

## 3. Где «табличка» и где «рабочие»?

| Простыми словами | Где в проекте (путь от корня `www/`) |
|------------------|--------------------------------------|
| «Кто отвечает за `/api/...`» | `local/php_interface/legacy/api.php` |
| «Таблица: какой адрес → какой класс» | `local/php_interface/legacy/classes/Api/ApiRoutes.php` |
| «Создать нужный класс по таблице» | `local/php_interface/legacy/classes/Api/ApiFactory.php` (обычно не трогают) |
| «Один конкретный ответ, например новости» | папка `.../Api/Methods/` — там файлы вроде `News.php` |

Ты **почти всегда** трогаешь только:

- **`ApiRoutes.php`** — добавить или поменять **строку маршрута**;
- **новый файл в `Methods/`** — написать **логику** в методе `init()`.

---

## 4. Как «добавить свой API» — три шага без магии

**Шаг 1.** Придумай **адрес**, например: данные по заявкам → **`/api/leads/list`**.

**Шаг 2.** Создай **PHP-класс** (новый файл), который **наследует** общий базовый класс API и в методе **`init()`** заполняет данные — например список заявок из инфоблока или заглушку-массив для теста.

**Шаг 3.** Открой **`ApiRoutes.php`** и добавь **одну строку**:  
`'/api/leads/list' => ... твой класс ...`

Сохрани файлы, открой в браузере **`/api/leads/list`** — должен прийти JSON.

Если **ничего не поменялось** — чаще всего: опечатка в пути, класс не в том namespace, или кэш/правила Битрикс; тогда смотри [07-legacy-api.md](07-legacy-api.md) или спроси наставника.

---

## 5. Как «изменить» уже существующий API

Найди в **`ApiRoutes.php`** нужную строку (например `/api/news`) — рядом указано **имя класса**.

Открой **соответствующий файл** в `Methods/` (например `News.php`) и **меняй только логику внутри `init()`**: что класть в ответ, откуда брать данные, какие проверки делать.

**Менять адрес** (`/api/news` → другое имя) = поменять **и** строку в `ApiRoutes`, **и** всех, кто на старом адресе уже обращается (фронт, Postman, интеграции).

---

## 6. Что такое JSON и зачем он тут

**JSON** — это текст в виде «фигурные скобки, кавычки, запятые». Его легко прочитать и **человеку**, и **программе**. В ответе почти всегда есть поле вроде **`success`**: правда/ложь — получилось или ошибка.

Не нужно знать JSON наизусть: открой ответ в браузере или в Postman — увидишь структуру.

---

## 7. Что пока можно не учить глубоко

- Как именно Битрикс пишет правило **urlrewrite** в базу — это уже сделано **автоматически** при первом запуске логики сайта.
- Как устроен **пролог** и **epilog** — пока достаточно знать: «до ответа API подключён Битрикс, можно пользоваться его функциями и базой».

Когда подрастёшь по уровню — возвращайся к [07-legacy-api.md](07-legacy-api.md): там всё по полочкам для разработчика.

---

## 8. Мини-памятка одной строкой

**Запрос `/api/что-то` → смотрим таблицу маршрутов → вызываем класс → в `init()` собираем ответ → отдаём JSON.**

Если запутался — нарисуй на бумаге стрелку: **URL → api.php → ApiRoutes → твой класс → JSON**.
